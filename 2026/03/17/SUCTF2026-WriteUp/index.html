SU_babyAI

It seems like something is missing.

在深度学习中，如果没有类似 ReLU、Sigmoid 这样的非线性激活函数，无论叠加多少层卷积层 (Conv1d) 或全连接层 (Linear)，并且 bias=False，其正向传播的本质都可以完全坍缩、折叠成一次普通的矩阵乘法。我们可以将这两层网络等效为一个二维系数矩阵 M15×41M_{15 \times 41}M15×41​

对于输入列向量 XXX（即 Flag 的 ASCII 码），网络的前向传播等价于：$$ Output = M \times X $$

观察源码最后的输出处理：

每次输出都加上了 [−160,160][-160, 160][−160,160] 范围的小噪音，并对大素数 q=1000000007q = 1000000007q=1000000007 取模，得到最终的数组 Y15Y_{15}Y15​，也就是 LWE

Y≡M×X+E(modq)Y \equiv M \times X + E \pmod q Y≡M×X+E(modq)

MMM 是我们通过网络权重推导出的 15×4115 \times 4115×41 等效系数矩阵
XXX 是 41 个字符的 FLAG。已知 SUCTF{}，实际未知数为 34 个。
EEE 是范围在 [−160,160][-160, 160][−160,160] 之间的噪音向量

剔除噪音 EEE，需要利用 Kannan 嵌入法 (Kannan's Embedding) 将 LWE 转化为最近向量问题 (CVP) 构建晶格，并通过 LLL 算法 寻找最短向量（SVP）

为了确保 LLL 算法 100% 收敛找到最短向量，必须运用两个关键的格优化技巧：
中心平移 (Center Shift)：将未知数 XXX 从 [32,126][32, 126][32,126] 平移到 [−47,47][-47, 47][−47,47]（减去 ASCII 中心值 79），极大降低目标向量的范数

方差配平 (Weight Balancing)：平移后的未知数最大约 47，而噪音最大为 160。我们在对角矩阵中将未知数权重放大 3 倍（3×47≈1413 \times 47 \approx 1413×47≈141），使其在格中的地位（方差）与噪音相近。

SU_theif

老师布置了一道课后作业，小S很早就写完了并且部署好了，小U很懒惰，不想写想直接抄小S的，但是小S不让，所以请你帮帮他，注意了这件事情不合规所以不要和任何人说

关键漏洞在：

  • 服务端只校验了 2 维参数和 1 维参数（也就是 linear.weight 和各 bias）
  • 4 维卷积权重完全没有被检查（conv.weight、conv1.weight）

直接提交 model_base.pth 会失败，报第 0 层（linear.weight）差异过大。所以正确做法是：

  1. 保留 base 模型的卷积层
  2. 调用 /predict 收集随机输入 x 与输出 y
  3. 用 base 卷积层把 x 映射成特征 z
  4. 通过最小二乘一次性解出线性层参数：y ≈ Wz + b
  5. 把求得的 W、b 写回模型后提交 /flag

SU_Restaurant

Well, aren’t there a bit too many cryptography challenges here?

题目实现了一个类似于热带半环数字签名的协议，但我们可以通过巧妙构造矩阵，在完全不知道密钥（即代码中的 fork 或 F）的情况下伪造合法的签名

在代码自定义的 Point 和 Block 类中，重载了加法和乘法：
加法 (+)：取两个数的最小值 min⁡(x,y)\min(x, y)min(x,y)
乘法 (*)：取两个数的一般加法 x+yx + yx+y

在验证逻辑 eat() 中，服务器要求我们提供 A,B,P,R,SA, B, P, R, SA,B,P,R,S 五个矩阵，使得：$$W = A \times B = M \times F \times M + M \times P + R \times M + S = Z$$且满足 W≠SW \neq SW=S 以及所有矩阵为满秩（常规线性代数意义下的秩）。因为加法是取最小值，上述等式可以转换为标准数学表达：$$Z_{i,j} = \min(MFM_{i,j}, MP_{i,j}, RM_{i,j}, S_{i,j})$$漏洞点：虽然我们不知道 FFF，但我们可以通过控制 WWW 和其余矩阵，让 ZZZ 的最小值完全不依赖于 MFMMFMMFM。由于 MFMi,j=min⁡k,l(Mi,k+Fk,l+Ml,j)MFM_{i,j} = \min_{k,l} (M_{i,k} + F_{k,l} + M_{l,j})MFMi,j​=mink,l​(Mi,k​+Fk,l​+Ml,j​)，且 F≥0F \ge 0F≥0，因此我们有一个硬性下界：$$MFM_{i,j} \ge \min_k M_{i,k} + \min_l M_{l,j}$$只要我们构造的 Wi,j=min⁡kMi,k+min⁡lMl,jW_{i,j} = \min_k M_{i,k} + \min_l M_{l,j}Wi,j​=mink​Mi,k​+minl​Ml,j​，就能保证 W≤MFMW \le MFMW≤MFM 永远成立。此时，只要让 MPMPMP 或 RMRMRM 等于 WWW，就能在不知道 FFF 的情况下完美控制 ZZZ

设 MMM 的第 iii 行最小值为 R_miniR\_min_iR_mini​，第 jjj 列最小值为 C_minjC\_min_jC_minj​。我们将目标矩阵设为 Wi,j=R_mini+C_minjW_{i,j} = R\_min_i + C\_min_jWi,j​=R_mini​+C_minj​

构造 AAA 和 BBB：我们需要 A×B=WA \times B = WA×B=W，且 A,BA, BA,B 满足常规秩要求（秩为 7）。我们可以利用随机的布尔覆盖矩阵：如果覆盖位为 1，则填入 R_miniR\_min_iR_mini​ 或 C_minjC\_min_jC_minj​；否则填入较大的随机数（如 200）。这样既能保证热带乘积为 WWW，又能保证满秩

SU_Thief

The lazy admin neglected that the closest thief around him could help him steal /root/flag

可以发现是 Grafana 11.0.0，尝试默认账密没有进去，然后爆了一手是admin/1q2w3e，结合版本可以搜到 CVE-2024-9264

虽然 CVE-2024-9264 能 RCE，但执行身份是 grafana，权限不足以读取 /root/flag

所以这题的重点不是“直接打 RCE 读 root”，而是题目提示里的closest thief around him could help him steal

枚举进程后发现：

  • caddy 以 root 身份运行
  • 本地可访问 Caddy admin API：127.0.0.1:2019

也就是：

  1. 我们先用 Grafana RCE（grafana 权限）
  2. 再从内网访问 Caddy admin API（root 服务）
  3. 热加载一个恶意 Caddy 配置，让它把 / 当静态目录暴露到一个新路由
  4. 最后从外部 HTTP 直接读 /root/flag
